# From Research (01-01)
Architecture:
- Cloudflare Worker with Hyperdrive for RDS connection
- node-postgres (pg) v8.16.3+ 
- KV namespace for job caching
- Node.js compatibility required

API Endpoints:
- GET /jobs - List all jobs (cached)
- GET /jobs/:id - Single job detail
- POST /jobs/refresh - Full cache reload (last 7 days only)
- POST /jobs/sync?since=timestamp - Incremental sync (updated only)

Caching:
- KV key "jobs:all" - all active jobs JSON
- KV key "jobs:last_sync" - last sync ISO timestamp
- **7-day TTL on jobs data** (cost optimization per user requirements)

**COST OPTIMIZATION (per user requirements):**
- Data stored for 7 days (CACHE_TTL = 604800)
- Only fetch last 7 days: `WHERE created_at > NOW() - INTERVAL '7 days'`
- Incremental sync: `WHERE updated_at > last_sync_timestamp`
</context>
