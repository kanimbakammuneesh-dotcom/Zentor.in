  <action>
Caching strategy:
1. On first request: fetch all jobs from RDS, store in KV with 1-hour TTL
2. GET /jobs returns cached data (instant response)
3. GET /jobs/sync?since={last_sync} queries WHERE last_updated > since
4. Manual trigger: POST /jobs/refresh clears cache, fetches fresh
5. Background: Every 1 hour, cron triggers refresh

KV key structure:
- "jobs:all" - all active jobs JSON
- "jobs:last_sync" - ISO timestamp of last sync
- "jobs:count" - total count for pagination
</action>
